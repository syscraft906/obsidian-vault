# Security Advisory: Addressing Recent Vulnerabilities in Angular

## AI Summary
This Angular blog post addresses two critical security vulnerabilities in Angular SSR: Server-Side Request Forgery (SSRF) with header injection and open redirect issues via X-Forwarded-Prefix headers. It explains the risks, provides steps to update to patched versions, and offers temporary workarounds like header validation and sanitization. Developers using SSR are urged to update immediately, while non-SSR users are advised to maintain the latest supported versions.
