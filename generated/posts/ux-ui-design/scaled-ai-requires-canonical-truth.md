# Scaled AI Requires Canonical Truth

## AI Summary
This post emphasizes the necessity of establishing a single, authoritative source of truth (canonical knowledge) for enterprise AI deployments. It explains how large language models' probabilistic nature conflicts with deterministic enterprise requirements and proposes a three-layer solution: canonical knowledge, semantic translation, and AI as an interface to backend systems. The discussion highlights data governance, semantic layers, and practical analogies to clarify these non-negotiable infrastructure needs for scalable AI.
